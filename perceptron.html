PERCEPTRON

A study note

John C.

March 2026

Word on the street is that AI will be the end of humanity (which I suppose is one way it solves all our problems), so I figured my remaining days are best spent understanding the cause of my inevitable demise and/or transcendence. And what better way to study the nature of intelligence than writing a few lines of code and playing around with toy datasets? No algorithm seemed better suited as a starting point than perceptrons, "the simplest learning machines" as Minsky and Papert (1969) call them.

Of course, I immediately faced a roadblock: What exactly is a perceptron? Textbooks cite Rosenblatt (1957) as its origin, yet they define it in ways that depart significantly from his original work. Perhaps this is because Rosenblatt was building a physical device (his report includes circuit diagrams), the architecture and nomenclature of which did not endure. But even the formal, more abstract definition in Rosenblatt (1962) — "a network of... units with a variable interaction matrix" — sounds quite different from the simple "textbook" perceptron.

Unwisely, I decided to dig deeper. What I found was mostly layers of confusion and obscurity, much like Dante's Hell or a multilayer perceptron. The term perceptron can describe a device, an optimization algorithm, a class of directed graphs, or a real function. Nilsson (1965), for instance, describes it as "a two-layer machine," whereas Nilsson (1998), yes, the same author, treats it as a synonym for a simple function called the threshold logic unit.

The journey wasn't entirely fruitless, for it yielded some kind of framework that helped me understand different ideas in machine learning, including variants of perceptrons such as PAPA (Gamba, 1961) and voted perceptron (Freund & Schapire, 1999).

This note shares some of what I learned. The scope is limited to a deterministic setting where output labels are determined by some function, with most examples using binary output. I am no doubt oversimplifying or outright wrong about some concepts here, but such is the nature of study notes.

    Preliminaries

    I establish some terminology around datasets and discuss a deterministic data generating process that I assume throughout the post. Then I ramble a bit about inductive reasoning.

    Datasets

    Given a pair of sets \((\mathcal{X}, \mathcal{Y})\), a dataset is an element of \( (\mathcal{X} \times \mathcal{Y})^N \) for some positive integer \(N\). I'll say

    • \(\mathcal{X}\) is the input space,
    • \(\mathcal{Y}\) is the output space,
    • the \(i\)th coordinate \((x_i, y_i)\) of a dataset is its \(i\)th observation, and
    • \(N\) is the number of observations.

    It is often useful to treat a dataset as an element of \(\mathcal{X}^N \times \mathcal{Y}^N\), so that a dataset is compactly written as \((X, y)\), where

    • \( X := (x_1, \dots, x_N) \in \mathcal{X}^N\) is the input data, and
    • \( y := (y_1, \dots, y_N) \in \mathcal{Y}^N\) is the output data.

    Numeric datasets

    Typically, the output is a real number, \( \mathcal{Y} \subseteq \R \), in which case I treat the output data \(y \in \R^N\) as a column vector. In many applications, the output space will be a finite set of integers representing categories of the inputs.

    Similarly, the input space \(\mathcal{X}\) typically consists of \(K\)-tuples of real numbers, \(\mathcal{X} \subseteq \R^K\), in which case I call \(K\) the number of input variables and write \(x_i = (x_{i,1}, \dots, x_{i,K})\), treated as a row vector. This way, \(X = (x_1, \dots, x_N)\) can be seen as an \(N \times K\) matrix, where \(x_i\) is the \(i\)th row of \(X\), and \(x_{i, k}\) is the \(i\)th row \(k\)th column entry of \(X\).

    Deterministic data generation

    Consider a dataset whose output \(y_i\) for each observation \(i\) is determined by some function \(f^*: \mathcal{X} \to \mathcal{Y}\), which I refer to as the data generating function. There seem to be many names for this function. From the perspective of estimation and learning, it is the target function, ground truth, supervisor, or oracle. In computational learning theory, it is called a concept, although I find the term confusing when the output value is not Boolean. I am inclined to call it a pattern, so phrases like "learn the underlying pattern of the dataset" become sensible. However, Duda et al. (2000) use the term pattern to refer to an input value \(x \in \mathcal{X}\). More precisely, given a function \(f^*: \mathcal{X} \to \mathcal{Y}\) and some input data \(X \in \mathcal{X}^N\), we can derive the output data as \(y = (f^*(x_1), \dots, f^*(x_N))\) to complete the dataset \((X, y)\). Denote such a dataset by \[ D(X, f^*) := \left( (x_1, f^*(x_1)), \dots, (x_N, f^*(x_N)) \right). \]

    For example, Rosenblatt (1957) considers a dataset where each input \(x_i\) is a black-and-white image of either a circle or a square, and \(f^*\) is an indicator that \(x_i\) is an image of a circle. Another example from Minsky and Papert (1969) is a dataset where each input \(x_i\) is a finite set, and \(f^*\) is an indicator that \(x_i\) has an odd number of elements.

    The following 'family' notations will be useful:

    • Let \(F(\mathcal{X}, \mathcal{Y})\) denote the set of all functions with domain \(\mathcal{X}\) and codomain \(\mathcal{Y}\): \[ F(\mathcal{X}, \mathcal{Y}) := \left\{f : \mathcal{X} \to \mathcal{Y}\right\}. \]
    • Let \(\mathcal{D}(\mathcal{X}, \mathcal{Y})\) denote the set of all datasets \(D(X, f^*)\) such that \(X \in \mathcal{X}^N\) for some positive integer \(N\) and \(f^* \in F(\mathcal{X}, \mathcal{Y})\). We can relax the assumption that \(f^*\) is any function of the given input and output spaces and instead specify a set of functions (i.e. a subset of \(F(\mathcal{X}, \mathcal{Y})\), called the concept class) from which the data generating function can be selected.

    Drawing datasets

    We have characterized datasets \(D(X, f^*)\) whose output values are determined by a data generating function \(f^*\). This raises the question of how the input data \(X\) are chosen from \(\mathcal{X}^N\), and how the data generating function \(f^*\) is chosen from \(F(\mathcal{X}, \mathcal{Y})\).

    A common assumption about the input data \(X\) is that each input observation \(x_i\) is drawn from \(\mathcal{X}\) according to some common distribution \(G\). In other words, \(X = (x_1, \dots, x_N)\) is a realization of an \(N\)-tuple of \(\mathcal{X}\)-valued iid random variables.

    Under this iid assumption, the data generating process (not to be confused with the data generating function) is fully described by \((f^*, G, N)\) where

    • \(f^* : \mathcal{X} \to \mathcal{Y}\) is a data generating function,
    • \(G\) is the distribution over \(\mathcal{X}\) that determines the draw of each input observation, and
    • \(N\) is the number of observations.

    What about the choice of the data generating function \(f^*\)? Ideally, we want to stay agnostic and cover a wide range of data spaces. So we will see how far we can get without further assumptions.

    Problem of induction

    The learner's task is to figure out the data generating function \(f^*\) given a dataset \(D(X, f^*)\).

    If the given input data \(X\) covers the entire input space \(\mathcal{X}\), then the task is trivial; the given dataset fully describes the function \(f^*\).

    The challenge arises when \(X\) covers only a portion of the input space. The problem is that of inductive generalization: to draw a conclusion about the whole function \(f^*\) based on a sample of its evaluations.

    No free lunch

    We face an immediate problem with our agnostic approach to data generation. To illustrate the problem, consider a dataset with input and output spaces \(\mathcal{X} = \mathcal{Y} = \{1, \dots, 10\}\) and the observations \(\{(1, 1), (2, 2), (4, 4), \dots, (10, 10)\}\). The dataset is plotted in .

    Given this dataset, the only thing we don't know about the data generating function \(f^*\) is its evaluation at \(x = 3\). The figure shows two hypotheses. Under hypothesis A, \(f^*_A(3) = 3\), so the hypothesized data generating function is \(f^*_A(x) = x\). Under hypothesis B, \(f^*_B(3) = 7\), so that \[f^*_B(x) = \begin{cases} x & \text{if \(x \neq 3\)}, \\ 7 & \text{if \(x = 3\)}. \end{cases} \]

    Which of these two hypotheses is more likely to be the true data generating function? If we translate our agnosticism about \(f^*\) into a uniform distribution over all functions \(F(\mathcal{X}, \mathcal{Y})\), then the answer is that both are equally likely, each having a 10% chance of being the true data generating function conditional on the dataset.

    Occam's razor

    My brain (and presumably yours) sees that hypothesis A fits the "pattern" while hypothesis B does not. Clearly, the "straight line" function \(f^*_A\) is special in a way that \(f^*_B\) is not, so that we prefer it as our prediction even without knowing what quantities \(x\) and \(y\) represent. In other words, our induction is biased toward some hypotheses over others.

    What makes the function \(f^*_A\) preferable to \(f^*_B\)? The usual answer invokes Occam's razor: hypothesis A is preferable because it is simpler. This raises the question of whether and why simpler hypotheses should be preferred, but suppose we are willing to put aside the question and take the principle as given. We see a glimpse of a general learning machine: it imposes a probability distribution over \(F(\mathcal{X}, \mathcal{Y})\) such that more complex functions are less likely, and updates the distribution by Bayes' rule when given a dataset.

    Alas, this induction procedure has seen little practical application. While a broad notion of complexity that applies to arbitrary functions does exist, it is not computable.

    An alternative approach restricts the set of permitted hypotheses. This breaks the uniform prior by assigning zero probability to prohibited hypotheses. It also lets us define a "customized" notion of simplicity tailored to the permitted hypotheses. This approach motivates the following definition of the perceptron.

    Perceptron

    Given the input and output spaces \((\mathcal{X}, \mathcal{Y})\), a perceptron is a pair \((H, \phi)\), where

    • \(H \subseteq F(\mathcal{X}, \mathcal{Y})\) is called the hypothesis space, and
    • \(\phi: \mathcal{D}(\mathcal{X}, \mathcal{Y}) \to H\) is the learning algorithm that, given a dataset, chooses a function from the hypothesis space.

    Many modern textbooks define the perceptron much more narrowly, with the hypothesis space involving a linear aggregation and the learning algorithm involving iterative error correction. (I call this the §simple perceptron.) But Rosenblatt (1957) himself provides a broad definition (predating Samuel's (1959) introduction of the term "machine learning"):

    ...it should be feasible to construct an electronic or electromechanical system which will learn to recognize similarities or identities between patterns of optical, electrical, or tonal information, in a manner which may be closely analogous to the perceptual processes of a biological brain. The proposed system... gains its reliability from the properties of statistical measurements obtained from large populations of elements. A system which operates according to these principles will be called a perceptron.

    While general, the definition is not without bite, in that perceptrons can learn only through data. No new knowledge is gained from thinking alone. The definition also requires the learner to choose a hypothesis from a given pool rather than constructing new hypotheses based on observations. It is difficult not to suspect that these constraints limit what perceptrons can learn, but maybe I will be proven wrong.

    Consistent hypotheses

    Missing from the definition is any criterion for the learning algorithm. Informally, we want the perceptron's learning algorithm \(\phi\) to pick a function from the hypothesis space that is similar to the actual data generating function: \(\phi(D(X, f^*)) \approx f^*\). We can and will make precise the gap between a hypothesis and the data generating function. We could then think about the distribution of that gap given the probability distribution of datasets.

    Even without wading through formality, it would be a no-brainer for the learning algorithm to choose a hypothesis that matches the given dataset. If we know from the dataset that \(f^*(x_i) = y_i\) for the observed \(x_i \in \mathcal{X}\), why would we choose a hypothesis that says otherwise?

    To be more precise, we say a hypothesis \(h \in H\) is consistent with a dataset \((X, y)\) if \(h(x_i) = y_i\) for all \(i = 1, \dots, N\). (Not to be confused with consistency of an estimator.)

    There are two remaining problems. One is that a consistent hypothesis may not exist in the given hypothesis space, which is always possible unless the hypothesis space is universal. Addressing this problem is not too difficult. We can generalize the notion of consistency and measure how well a hypothesis fits the given dataset. Then we can simply choose the best-fitting hypothesis. The principle of §empirical risk minimization formalizes this idea.

    The other problem is harder: there may be more than one consistent hypothesis. Notice this situation is more likely if the dataset is smaller or if the hypothesis space is broader. In this case, the learning algorithm would need a way to pick a hypothesis from many and essentially provide its own inductive bias.

    Risk

    To evaluate perceptrons, we want a more precise way to measure how well a chosen hypothesis \(h\) matches the data generating function \(f^*\). To do so, we can look at their gap \(f^*(x) - h(x)\) at each input value \(x \in \mathcal{X}\), measure the size of that gap, and take an appropriately weighted sum across all input values. The details below introduce the machine learning lingo associated with this procedure.

    Throughout this section, assume numerical datasets, so that \(\mathcal{Y} \subseteq \R\) and \(\mathcal{X} \subseteq \R^K\). Furthermore, assume an iid data generating process \((f^*, G)\). Then the distribution of an \(\mathcal{X}\)-valued random variable is fully determined by its cdf. As such, we let \(G\) denote the cdf of the input observation.

    Error

    Define the error of a hypothesis \(h \in H\) at some input value \(x \in \mathcal{X}\) as the difference between the true and the hypothesized output value at \(x\): \[ \epsilon(x; h) := f^*(x) - h(x). \]

    Loss

    The loss function \(L: \R \to \R_+\) is a non-negative transformation of error with \(L(0) = 0\), capturing the size of the error. The literature defines loss function as \(L : \R^2 \to \R_+\), taking as arguments the true output value and the predicted value. This in effect allows a broader definition of errors. Examples include:

    • 0-1 loss, \(L(\epsilon) = \ind(\epsilon \neq 0)\),
    • absolute loss, \(L(\epsilon) = |\epsilon|\), and
    • square loss, \(L(\epsilon) = \epsilon^2\).

    Which loss function to use? The answer seems to be whichever one is easy to work with. That depends on the context, so choosing the right loss function seems to be something of an art.

    Risk

    Since error is defined locally at a given input value \(x \in \mathcal{X}\), we need an aggregation scheme to evaluate the error (and loss) of a hypothesis as a whole. A sensible scheme would take the sum weighted by the probability of drawing the input value, i.e. the expected value.

    The expected value of the loss of a given hypothesis is called its risk: \[ R(h) := \E\left[ L(\epsilon(\xi; h)) \right] = \int_{\mathcal{X}} L(f^*(x) - h(x)) \dd{G(x)}, \] where \(\xi\) is a random variable with distribution \(G\).

    Estimating risk

    Since the data generating function \(f^*\) and distribution \(G\) are unknown, we cannot calculate the risk directly. We can, however, estimate it as the sample mean of the loss.

    Given some dataset \((X, y)\) and a hypothesis \(h \in H\), define the error on the \(i\)th observation as \[ e_i(h) := y_i - h(x_i). \] (Under the deterministic data generation process we've assumed, \(e_i(h) = \epsilon(x_i; h)\) since \(y_i = f^*(x_i)\).)

    The risk of a hypothesis \(h\) can be estimated from a dataset by \[\begin{equation}\label{eq:risk_sample_mean} \hat{R}(h) := \frac{1}{N} \sum_{i=1}^N L(e_i(h)). \end{equation}\]

    If the hypothesis under consideration \(h\) is the evaluation of a learning algorithm on some dataset \((X, y) = D(X, f^*)\), so that \(h = \phi(D(X, f^*))\), then an unbiased risk estimate requires a separate, independent dataset \((X', y') = D(X', f^*)\) where each observation in \(X'\) is drawn from the same distribution \(G\). To make the distinction, the dataset \(D(X, f^*)\) used by the learning algorithm is called the training dataset, while the dataset \(D(X', f^*)\) used to estimate the risk is called the testing dataset.

    Hypothesis space

    Parametric hypothesis space

    The predominant way to specify the hypothesis space \(H\) is as the set of functions obtained by slicing a "hypothesis generating function" \[ h : \mathcal{X} \times \Theta \to \mathcal{Y}. \] Here, \(\Theta \subseteq \R^S\) is called the parameter space. Each hypothesis in the space generated by \(h\) is the function \(h_\theta : \mathcal{X} \to \mathcal{Y}\) defined by \(h_\theta(x_i) = h(x_i, \theta)\).

    For example, if \(\mathcal{X} = \mathcal{Y} = \R\), then the function \(h : \R \times \R^2 \to \R\) defined by \(h(x, \theta) := \theta_0 + \theta_1 x\) generates the hypothesis space of all affine functions in \(F(\R, \R)\).

    I will at times say "hypothesis \(\theta\)" for a parameter \(\theta \in \Theta\) to mean the corresponding hypothesis \(h_\theta \in H\). Since the mapping from \(\theta\) to \(h_\theta\) is by construction surjective, the property that holds for all parameters \(\theta \in \Theta\) holds for all hypotheses \(h \in H\) as well.

    Estimation and approximation risks

    The following decomposition of risk is useful to structure the discussion of hypothesis spaces. Consider the minimum possible risk achievable under some hypothesis space \(H\): \[ R^*(H) := \inf_{h \in H} \; R(h). \] We can decompose the risk associated with the hypothesis chosen by some learning algorithm \(h = \phi(D(X, f^*))\) as \[\begin{equation}\label{eq:risk_decomp} R(h) = \underbrace{\left( R(h) - R^*(H) \right)}_{\text{estimation}} + \underbrace{R^*(H).}_{\text{approximation}} \end{equation}\] Note each summand, labeled estimation and approximation, is non-negative.

    Approximation risk depends solely on the hypothesis space \(H\) and generally grows as the space becomes more restrictive. If the true data generating function \(f^*\) lies in the hypothesis space, then this term is zero. Otherwise, its magnitude depends on how well one of the hypotheses in \(H\) can approximate \(f^*\).

    Since restricting the hypothesis space generally increases approximation risk, it reduces the overall risk only if the corresponding reduction in estimation risk more than offsets. The only way to reduce estimation risk is indirectly through \(\phi\). (The decomposition is pointless if \(h\) is fixed.) In other words, restrictions on the hypothesis space are helpful only if they allow the learning algorithm to perform better.

    Restrictions on the hypothesis space can help the learning algorithm by introducing inductive bias in two ways. First, they break the "uniform" prior by assigning zero probability to some functions. We can apply Occam's razor by simply excluding "complex" functions as candidates.

    Second, they make inductive bias easier to implement. The key challenge with a universal prior is the difficulty of ranking simplicity among arbitrary functions. This becomes easier when we only need to rank functions within a specific class. For example, if we restrict the hypothesis space to polynomials (among all real functions), the degree of a polynomial serves as an intuitive and simple (albeit coarse) measure of complexity.

    In short, a successful hypothesis space specification is sufficiently flexible to limit approximation risk, and sufficiently structured to yield an effective complexity measure.

    Example: Calculating risk components

    Consider again the example illustrated in . Suppose we consider the hypothesis space \(H\) consisting of all affine functions, which we can specify parametrically with the hypothesis generating function \(h(x; \theta) := \theta_0 + \theta_1 x\), where \(x \in \mathcal{X} = [0,1]\) and \(\theta \in \Theta = \R^2\).

    Continuing with the absolute loss, the approximation risk is \[ R^*(H) = \inf_{\theta \in \R^2} \int_0^1 |(4x^3 - 6x^2 + 3x) - (\theta_0 + \theta_1 x)| \dd{x} = 0.0625 \] achieved at \(\theta = (0.25, 0.5)\). For the example hypothesis \((0.2, 0.6)\), which has the risk of \(R(h) = 0.065\), the estimation risk is \(0.065 - 0.0625 = 0.0025\).

    On the other hand, if the hypothesis space \(H\) includes all polynomials of order up to 3, then \(f^* \in H\), so the approximation risk is zero, and the estimation risk of the example hypothesis \(h\) is \(0.065\).

    Empirical risk minimization

    If we use the risk \(R(h)\) as our performance metric, a natural approach is to minimize its sample analog. This leads to the principle of empirical risk minimization (ERM): given a hypothesis space \(H\) and a training dataset \(D(X, f^*)\), we choose a hypothesis from \(H\) that minimizes the empirical risk: \[\begin{equation}\label{eq:erm} \min_{h \in H} \; \hat{R}(h), \end{equation}\] where \(\hat{R}\) is defined in \(\eqref{eq:risk_sample_mean}\) but using the training rather than testing dataset.

    A technical issue arises here: if \(H\) is infinite, the minimum may not exist. We could avoid the issue by saying an algorithm follows the ERM principle if, for a given threshold \(\epsilon > 0\), it chooses a hypothesis that yields an empirical risk within \(\epsilon\) of the infimum.

    This formalism seems excessive here, so I will assume the infimum is attained and use equality. Specifically, if \(\phi\) follows the ERM principle, then the chosen hypothesis \(\hat{h} = \phi(D(X, f^*))\) satisfies \[ \hat{R}(\hat{h}) = \inf_{h \in H} \; \hat{R}(h). \]

    Example: ERM with square loss

    Suppose the hypothesis space is parameterized by \(h : \mathcal{X} \times \Theta \to \mathcal{Y}\), where \(\Theta \subseteq \R^S\). Under the square loss function \(L(\epsilon) = \epsilon^2\), the ERM problem \(\eqref{eq:erm}\) becomes: \[ \min_{\theta \in \Theta} \; \frac{1}{N} \sum_{i=1}^N e_i^2(\theta), \quad \text{where } e_i(\theta) := y_i - h(x_i, \theta). \]

    If \(h\) is differentiable in \(\theta\), the first-order condition for a minimum is: \[ \sum_{i=1}^N e_i(\theta) \left(\frac{\partial h(x_i, \theta)}{\partial \theta_s} \right) = 0 \] for each \(s = 1, \dots, S\). In matrix form, \[\begin{equation}\label{eq:erm_foc} J^\mathsf{T} e = 0, \end{equation}\] where \(J\) is the \(N \times S\) matrix whose \((i, s)\)-entry is \(\frac{\partial h(x_i, \theta)}{\partial \theta_s}\), and \(e\) is the \(N\)-vector of residuals \(e_i(\theta)\).

    If \(h\) is additionally convex in \(\theta\), then the first order condition is also sufficient, so the equation \(\eqref{eq:erm_foc}\) characterizes the solution.

    For instance, if \(h : \R \to \R^{1+d}\) is a polynomial function, that is, \(h(x, \theta) = \theta_0 + \theta_1 x + \dots + \theta_d x^d\), then it is linear in \(\theta\), and the first order condition \(\eqref{eq:erm_foc}\) simplifies to a linear equation \[A \theta = b\] where \(A\) is a \((1+d)\) square matrix whose \((s, t)\)-entry, counting from 0, is \(A_{st} = \sum_{i=1}^N x_i^{s+t}\), and \(b\) is a \((1+d)\) vector whose \(s\)th entry, counting from 0, is \(\sum_{i=1}^N y_i x_i^s\). If \(h(x, \theta) = \theta \cdot (1, x, \dots, x^d)\), then \(\frac{\partial h(x, \theta)}{\partial \theta_s} = x^s\). The first order condition becomes \(\sum_{i=1}^N e_i(\theta) x_i^s = 0\) for each \(s = 0, \dots, d\). Expanding \(e_i(\theta) = y_i - \theta \cdot (1, x_i, \dots, x_i^d)\), the condition is \(\sum_{i=1}^N y_i x_i^s = \theta \cdot (\sum_{i=1}^N x_i^s, \dots, \sum_{i=1}^N x_i^{d+s})\) for each \(s\).

    Example: Polynomial fitting the Runge function

    If we follow the ERM principle, then the only remaining design decision is the choice of the hypothesis space. Looking back at the decomposition \(\eqref{eq:risk_decomp}\), we need to see how \(H\) balances the trade-off between estimation and approximation.

    Consider the example, due to Runge, that illustrates the potential tradeoff between estimation and approximation risks that comes with the choice of the hypothesis space. The setting involves an interval input space \(\mathcal{X} = [-1, 1]\) and the real output space \(\mathcal{Y} = \R\). The data generating function is given by \(f^*(x) := \frac{1}{1 + 25x^2}\), shown in with a dotted line, along with \(N = 14\) observations drawn from a uniform distribution.

    Let the hypothesis space \(H_k\) be the set of polynomials with degree at most \(k\). Since \(H_{k} \subseteq H_{k + 1}\), we must have \(R^*(H_k) \geq R^*(H_{k+1})\) (the best can only get better when you add more candidates). Nonetheless, the estimation risk may grow with \(k\) to the extent of raising the overall risk.

The risk of each hypothesis is decomposed into the estimation and approximation risks. As expected, the approximation risk falls with \(k\). With this particular dataset, the overall risk initially diminishes with \(k\) but starts to rise beyond \(k = 8\).

    To hint at the source of the rising estimation error, plots two of the ERM hypotheses, one for \(k = 4\) and other for \(k = 12\). As expected, the 12-degree polynomial hypothesis fits the observations better and is in fact closer to the true data generating function near the center of the domain. However, near the extremities of the domain, beyond the observation range, it deviates significantly, shooting up rather than tapering down. The flexibility of the hypothesis space, which allows low approximation error, permits an "unusual" behavior (perhaps unusual under the implicit prior distribution of the data generating functions) outside of the observed range, leading to high estimation error.

    Convergence of ERM

    Does the ERM estimation error go to zero as the number of observations grows? Not always. For example, consider the data generating process where

    • the input space is the unit interval, \(\mathcal{X} = [0, 1]\),
    • the output space is binary, \(\mathcal{Y} = \{0, 1\}\),
    • the data generating function is constant, \(f^*(x) = 1\), and
    • the input distribution is uniform, \(G \sim \text{Uniform}(0, 1)\).

    Consider a perceptron with the unrestricted hypothesis space \(H = F(\mathcal{X}, \mathcal{Y})\), implying zero approximation risk, and the learning algorithm that chooses the hypothesis \(h(x) = \ind(x \in X)\) for any given dataset \((X, y)\). That is, it predicts 1 if the input value is in the dataset but predicts 0 everywhere else. As perverse as this hypothesis seems, it is an ERM solution since it achieves zero empirical risk for any dataset. And yet, since it is correct only on finitely many points within the interval domain, its estimation risk is 1 regardless of the dataset. So the estimation risk does not diminish with the number of observations.

    Again, the problem is that the hypothesis space is too broad, allowing pathological hypotheses. On the other extreme, if the hypothesis space is a singleton, then the estimation risk (of any learning algorithm) would be trivially zero for any dataset.

    We can do a lot better than requiring a singleton hypothesis space to ensure the ERM estimation risk converges in probability to zero. Formally, we want to investigate the sufficient conditions on the hypothesis space \(H\) such that \(R(\hat{h}) \overset{p}\to R^*(H)\) as \(N \to \infty\) for any \(\hat{h}\) satisfying \(\hat{R}(\hat{h}) = \inf_{h \in H} \hat{R}(h)\).

    To parse through the notation, note that everything with a hat is random. Both the empirical risk \(\hat{R}\) and the ERM hypothesis \(\hat{h}\) are random since they depend on the training dataset which we are treating as random. Conversely, all hatless symbols denote non-random objects.

    Our strategy is to find a random variable \(\hat{Z}\) such that \(R(\hat{h}) - R^*(H) \leq \hat{Z}\), and determine a condition on \(\hat{Z}\) such that it converges in probability to zero. Since \(0 \leq R(\hat{h}) - R^*(H)\), the squeeze theorem applies, so the estimation risk converges in probability to zero if the upper bound \(\hat{Z}\) does.

    Bounding the estimation risk

    Since the ERM hypothesis is defined through \(\hat{R}\), the following decomposition of the estimation risk is useful: \[ R(\hat{h}) - R^*(H) = \underbrace{\left( R(\hat{h}) - \hat{R}(\hat{h}) \right)}_{(1)} + \underbrace{\left( \hat{R}(\hat{h}) - \hat{R}(h^*) \right)}_{(2)} + \underbrace{\left(\hat{R}(h^*) - R(h^*) \right)}_{(3)}, \] where \(h^* \in H\) is a hypothesis that satisfies \(R(h^*) = R^*(H)\).

    1. Let's start with the last term, \(\hat{R}(h^*) - R(h^*)\). This is simply the difference between the sample mean and the expected value of the loss evaluated at a best-in-class hypothesis \(h^*\). Given the iid assumption on the data generation, the standard law of large numbers applies, so the term converges in probability to zero.

    2. The second term is necessarily non-positive by definition since \(\hat{h}\) minimizes \(\hat{R}\).

    3. The first term, \(R(\hat{h}) - \hat{R}(\hat{h})\), is the difference between the true risk and the empirical risk evaluated at an ERM hypothesis \(\hat{h}\). Unlike the third term, the argument \(\hat{h}\) is random rather than fixed, so the "pointwise" convergence of \(\hat{R}\) from the law of large numbers is no longer sufficient. Even as the gap between \(\hat{R}\) and \(R\) shrinks at any given point, the point of evaluation \(\hat{h}\) may sneak away to another point with a larger gap.

      Since \(\hat{h}\) can go anywhere in \(H\), the upper bound strategy requires a blanket guarantee: \[R(\hat{h}) - \hat{R}(\hat{h}) \leq \sup_{h \in H} \left( R(h) - \hat{R}(h) \right).\]

    In summary, \[ R(\hat{h}) - R^*(H) \leq \sup_{h \in H} \left( R(h) - \hat{R}(h) \right) + \left(\hat{R}(h^*) - R(h^*) \right), \] and since the second summand converges in probability to zero by the law of large numbers, a sufficient Vapnik (1995, 1998), in what he modestly calls the "key theorem of learning theory," states that the uniform convergence is both sufficient and necessary for strict consistency. Strict consistency, in turn, is a condition stronger than consistency that excludes some trivial cases that arise with (regular) consistency. However, strict consistency also excludes non-trivial cases, as Shalev-Shwartz et al. (2010) point out: "uniform convergence is a sufficient, but not at all necessary, condition for consistency of empirical minimization in non-trivial settings." condition for the ERM estimation risk to converge to zero is that the empirical risk \(\hat{R}\) uniformly converges to \(R\): \[ \begin{equation}\label{eq:risk_uniform_convergence} \sup_{h \in H} \left( R(h) - \hat{R}(h) \right) \overset{p}{\to} 0. \end{equation} \]

    For example, if \(H\) is finite, then the uniform convergence necessarily holds (\(\hat{h}\) cannot keep running away forever). More broadly, Vapnik and Chervonenkis (1971) characterize a necessary and sufficient condition for uniform convergence when the output space is binary, \(\mathcal{Y} = \{0, 1\}\), and the risk is measured using absolute loss. In doing so, they introduce a property of hypothesis space, now called the VC dimension. Roughly speaking, it is the maximum number of observations for which there is a hypothesis in the space that perfectly fits the dataset for any data generating function \(f^*\). They show the uniform convergence holds if and only if the VC dimension of \(H\) is finite.

    The VC dimension is also useful for characterizing the rate of convergence. The first term of the estimation risk decomposition, \(R(\hat{h}) - \hat{R}(\hat{h})\), is bounded above by a term involving the probability the bound holds, the number of observations, and the VC dimension of the hypothesis space. The bound on the ERM hypothesis risk has been refined in subsequent works, Vapnik (1998) provides the following bound: \[ R(\hat{h}) < \hat{R}(\hat{h}) + \frac{\mathcal{E}(N, D, \eta)}{2} \left(1 + \sqrt{1 + \frac{4\hat{R}(\hat{h})}{\mathcal{E}(N, D, \eta)}}\right), \] where \[\quad \mathcal{E}(N, D, \eta) := 4 \frac{D(\ln(2N/D) +1) - \ln(\eta/4)}{N},\] \(N\) is the number of observations, \(D\) is the VC dimension of the hypothesis space, and \(\eta\) is the probability the bound fails. although taking no assumption on the data drawing distribution forces the worst-case scenario over the whole hypothesis space \(H\). Thus, treating this upper bound as an estimate for the actual risk doesn't seem to be a great idea, as the example below illustrates.

    Example: Polynomial hypothesis space

    If the hypothesis generating function \(h(x; \theta)\) is linear in \(\theta \in \R^S\), then the VC dimension of the hypothesis space is \(S\). Hence, the set of all polynomials degree at most \(d\) has a finite VC dimension of \(d + 1\).

    Let us continue from the previous example where the data generating function is the Runge function over the domain \(\mathcal{X} = [-1, 1]\), and the input data are drawn uniformly. We saw that for a small number of observations (\(N = 14\)), the ERM hypothesis from the 12-degree polynomial hypothesis space had a relatively large estimation risk.

    Simple perceptron

    In this section, I present a "textbook" perceptron, a version of Rosenblatt's perceptron simplified for pedagogy. It is defined for the binary outcome space, \(\mathcal{Y} = \{0, 1\}\), and a real vector input space, \(\mathcal{X} = \R^K\).

    Linear threshold hypothesis generating function

    The hypothesis space is specified parametrically, generated by a linear threshold function: \[\begin{equation}\label{eq:ltf} h(x_i, \theta) = \mathbf{1}( \theta \cdot \tilde{x}_i > 0), \end{equation}\] where

    • \(\mathbf{1}\) is the indicator function,
    • \(\tilde{x}_i = (1, x_{i,1}, \dots, x_{i,K})\), and
    • \(\theta \in \R^{1+K}\).

    Bias, weight, threshold, score

    It is useful to introduce some additional terminology. Given the parameter \(\theta = (\theta_0, \theta_1, \dots, \theta_K)\), let \(b := \theta_0\) and \(w := (\theta_1, \dots, \theta_K)\). Then \[\theta \cdot \tilde{x}_i = b + w \cdot x_i.\] The constant term \(b\) is called the bias (not to be confused with the bias of an estimator), and the other coefficients \(w\) are called the weights.

    If the inequality is written as \(\theta \cdot \tilde{x}_i > 0\), as in \(\eqref{eq:ltf}\), then \(\theta \cdot \tilde{x}_i = b + w \cdot x_i\) is called the score (not to be confused with the F-score). But if the same inequality is written as \(w \cdot x_i > -b\), then \(w \cdot x_i\) is called the score and \(-b\) the threshold. To avoid confusion, I will call \(\theta \cdot \tilde{x}_i\) the affine score and \(w \cdot x_{i}\) the linear score.

    Parameters and hypotheses

    The mapping \(\theta \mapsto h_\theta\) is not injective here. Indeed, if \(c > 0\) is some positive scalar, then \(h_\theta = h_{c \theta}\) for any \(\theta \in \R^{1+K}\). Fancily, we could say \(h(x_i, \theta)\) is positive homogeneous of degree zero in \(\theta\) to mean the same.

    The converse is almost true. If \(\theta \neq 0\) and \(h_\theta = h_{\theta'}\), then \(\theta' = c \theta\) for some \(c > 0\). If \(\theta = 0\), then \(h_\theta(x) = 0\) for all \(x \in \R^K\). So \(h_\theta = h_{\theta'}\) where \(\theta' = (b, 0, \dots, 0)\) for some \(b < 0\). But \(\theta \neq c \theta'\) for any \(c > 0\).

    In this note, I will say a parameter \(\theta\) is associated with a hypothesis \(h^*\) if \(h^* = h_\theta\), i.e. \(h^*(x) = h(x; \theta)\) for all \(x \in \mathcal{X}\).

    Error under linear threshold

    Lastly, the error term under the linear threshold hypothesis \(\theta\) is \[ e_i(\theta) := y_i - h(x_i, \theta) = \begin{cases} 1 &\text{if \(y_i = 1\) and \(\theta \cdot \tilde{x}_i \leq 0\)}, \\ -1 &\text{if \(y_i = 0\) and \(\theta \cdot \tilde{x}_i > 0\)}, \\ 0 &\text{otherwise}. \end{cases} \]

    Rosenblatt learning algorithm

    To find \(\theta\) that fits the given dataset, Rosenblatt (1958) proposes several variants of the algorithm sharing the following heuristics.

    Error correction

    The algorithm starts with an arbitrary guess of the hypothesis \(\theta_0 \in \R^{1+K}\) and updates it after examining each observation.

    1. Correct prediction. If \(e_i(\theta) = 0\), then the prediction is correct and there is no need to adjust \(\theta\).
    2. False negative. If \(e_i(\theta) = 1\), then \(y_i = 1\) and \(\theta \cdot \tilde{x}_i \leq 0\). To correct the error, we want to increase the value of \({\theta \cdot \tilde{x}_i}\), which we can achieve by incrementally moving \(\theta\) in the direction of the gradient \(\nabla (\theta \cdot \tilde{x}_i) = \tilde{x}_i\).
    3. False positive. Lastly, if \(e_i(\theta) = -1\), then we want to decrease \(\theta \cdot \tilde{x}_i\), which can be achieved by moving \(\theta\) in the opposite direction of the gradient, \(-\tilde{x}_i\).

    Conveniently, in all three cases, we want to update \(\theta\) in the direction of \(e_i(\theta) \tilde{x}_i\).

    Selection mechanism

    Repeated over the observations (each observation can be visited more than once), these updates result in a sequence of hypothesis parameters \((\theta_0, \theta_1, \dots)\). The learning algorithm outputs a parameter based on this sequence. If the sequence converges, then the limit provides a natural candidate. (See the §convergence theorem below.)

    In practice, the algorithm makes a predetermined number of updates and derives the parameter from the list \((\theta_0, \dots, \theta_T)\). One could simply choose the last parameter \(\theta_T\), or the one that minimizes the training dataset error rate. A more sophisticated selection mechanism, such as the voted perceptron proposed by Freund and Schapire (1999), takes a weighted average of the parameters in the sequence.

    Batch learning

    Instead of updating the hypothesis after each observation, the algorithm could update on a set of observations. That is, given any subset of observations \(B \subseteq \{1, \dots, N\}\), the algorithm adjusts the current candidate in the direction of \(\sum_{i \in B} e_i(\theta) \tilde{x}_i\). If \(B = \{1, \dots, N\}\), then this sum can be succinctly expressed as \(\tilde{X}^\mathsf{T} e(\theta) \), where \(\tilde{X}\) is an \(N \times (1 + K)\) matrix whose \(i\)th row is \(\tilde{x}_i\), and \(e(\theta)\) is an \(N\) vector whose \(i\)th coordinate is \(e_i(\theta)\).

    Below is one concrete version of the Rosenblatt algorithm.

    1. Given
      • a dataset \((X, y)\),
    2. Choose
      • the initial parameter \(\theta_0 \in \R^{1+K}\), such as \(\theta_0 = 0\),
      • the learning rate scalar \( \eta > 0 \), and
      • the length of the sequence \(T > 0\).
    3. Recursively define a sequence \( (\theta_1, \theta_2, \dots) \): \[ \theta_{t+1} := \theta_t + \eta \tilde{X}^\mathsf{T} e(\theta_t). \]
    4. Return \(\theta_T\).

    Convergence of Rosenblatt algorithm

    Linearly separable datasets

    Suppose \(\mathcal{Y} = \{0, 1\}\) and \(\mathcal{X} \subseteq \R^K\). We say a dataset \( (X, y) \in \mathcal{X}^N \times \mathcal{Y}^N \) is linearly separable if there exists some \(\theta^* \in \R^{1+K}\) such that \( y_i = h(x_i, \theta^*) \) for all \(i = 1, \dots, N\), where \(h\) is the linear threshold function \(\eqref{eq:ltf}\).

    Note that this does not imply the data generating function is the same linear threshold function, \(f^* = h_{\theta^*}\) for two reasons. One is due to sampling. Even if the data generating function is not a linear threshold function, the dataset generated by it can still be linearly separable. Indeed, if \(N = 2\), then the dataset is linearly separable regardless of its data generating function.

    Another reason is that even if \(f^*\) is a linear threshold function, its parameter need not be \(\theta^*\). In fact, with finitely many observations, there is always some "wiggle room" around the separating hyperplane, so a linearly separable dataset is separable by infinitely many \(\theta\)s. The result follows from the hyperplane separation theorem. The set \( \{x_i \in X : y_i = 1\} \) is finite, so its convex hull is compact, and so is the convex hull of \( \{x_i \in X : y_i = 0\} \). If the dataset is linearly separable, then these two convex hulls are disjoint. Combined with their compactness, the separation is strict. In other words, if \(y_i = h(x_i, \theta^*)\) for all \(i\), then there exists \(\theta' \neq \theta^*\) such that \(y_i = h(x_i, \theta')\) for all \(i\). illustrates a linearly separable dataset with two different consistent hypotheses.

    Perceptron convergence theorem

    The perceptron convergence theorem states that on a linearly separable dataset, the Rosenblatt algorithm converges in finitely many steps to some consistent hypothesis \(\theta'\). See, for example, Block (1962), Novikoff (1962), and Minsky and Papert (1969). More precisely, for any choices of \(\theta_0\) and \(\eta\), the sequence \((\theta_t)\) produced by the algorithm satisfies \(\theta_T = \theta'\) for some finite \(T\). Note that under \(\theta'\), all data points are correctly classified, so \( e(\theta_T) = 0 \). This in turn implies \(\theta_{T+1} = \theta_T\), so the sequence converges to \(\theta_T = \theta'\).

    Converse

    The converse of the convergence theorem also holds. If the dataset is not linearly separable (which implies \(f^*\) is not a linear threshold function), then the error vector \(e(\theta)\) is never zero for any \(\theta\), so it always gets updated. Since there are finitely many possible values for the adjustment \(\tilde{X}^\mathsf{T} e(\theta)\), the algorithm does not converge.

    Rosenblatt algorithm as ERM

    We can view the Rosenblatt algorithm as minimizing a type of empirical risk, an average of weighted errors, and therefore as following the principle of ERM. This framing helps to better understand the algorithm.

    Slack

    To motivate the weight, fix a dataset \((X, y)\) and a parameter \(\theta\). For an observation \(i\), consider the "slack" \(\xi_i(\theta)\) required on the threshold so that the score lies on the correct side of the slacked threshold.

    • If the observation is already correctly labeled, i.e. if \(e_i(\theta) = 0\), then there is no need for any slack, so \(\xi_i(\theta) = 0\).
    • If \(y_i = 0\) but \(\theta \cdot \tilde{x}_i > 0\), so that \(e_i(\theta) = -1\), then we need to raise the threshold so that \(\theta \cdot \tilde{x}_i \leq s\) for some \(s\). The minimum \(s\) such that the inequality holds is \(\xi_i(\theta) = \theta \cdot \tilde{x}_i\).
    • If \(y_i = 1\) but \(\theta \cdot \tilde{x}_i \leq 0\), so that \(e_i(\theta) = 1\), then we want the minimum \(s\) such that \(\theta \cdot \tilde{x}_i > -s\). Since the minimum does not exist with the strict inequality, however, we use the infimum, or equivalently, use weak inequality: \(-\theta \cdot \tilde{x}_i \leq s\). This gives us \(\xi_i(\theta) = -\theta \cdot \tilde{x}_i\).

    We can combine all three cases and write the slack succinctly as \[\xi_i(\theta) := (-\theta \cdot \tilde{x}_i) e_i(\theta). \]

    An alternative expression for the slack uses positive and negative output labels instead of one and zero. To this end, define \[ \tilde{y}_i := 2 y_i - 1 \in \{-1, 1\}. \] Then the slack inequality for either output value is \(s \geq \tilde{y}_i (-\theta \cdot \tilde{x}_i)\). Since the slack is by definition non-negative, \[ \xi_i(\theta) = \max\{0, -\tilde{y}_i (\theta \cdot \tilde{x}_i)\}. \]

    Perceptron criterion

    The Rosenblatt algorithm is a subgradient method that minimizes the average slack, or what Bishop (1995) calls the perceptron criterion: \[\begin{equation}\label{eq:perceptron_criterion} \bar{\xi}(\theta) := \frac{1}{N} \sum_{i=1}^N \xi_i(\theta) = \frac{1}{N} \sum_{i=1}^N (-\theta \cdot \tilde{x}_i) e_i(\theta). \end{equation}\]

    Viewed as a risk, the perceptron criterion's "loss" function is the weighted error \(L(x; \theta) = (-\theta \cdot \tilde{x}) \epsilon(x; \theta)\). Note \(L(x; \theta) \geq 0\) with equality when \(\epsilon(x; \theta) = 0\). Still, it is not a proper loss as I have defined it since the weight depends on the parameter value \(\theta\). Consequently, we cannot compare the performance of Rosenblatt algorithm using this risk to the performance of an algorithm under a different hypothesis space. Nonetheless, since it plays the role of a loss function in empirical risk minimization, it is sometimes called a surrogate loss function.

    The key advantage of using a surrogate rather than the proper loss is that it provides a nicer objective function: the average slack is continuous and convex, unlike the average error. Even though the error \(e_i(\theta) = y_i - h(x_i, \theta)\) by itself is discontinuous, the product \( (\theta \cdot \tilde{x}) h(x, \theta) = (\theta \cdot \tilde{x}) \ind(\theta \cdot \tilde{x} > 0) = \max\{0, \theta \cdot \tilde{x}\} \) is both continuous and convex in \(\theta\), and so is the empirical risk \( N\hat{R}(\theta) = \sum_{i=1}^N \max\{0, \theta \cdot \tilde{x}_i\} - \sum_{i=1}^N (\theta \cdot \tilde{x}_i) y_i\).

    Except at the points \(\theta \in \R^{1+K}\) such that \(\theta \cdot \tilde{x}_i = 0\) for some \(i\), the gradient of \(\bar{\xi}\) exists and is easily computed as \[\nabla \bar{\xi}(\theta) = -\frac{1}{N} \tilde{X}^\mathsf{T} e(\theta).\] One can also show that at any non-differentiable point \(\theta_0\), \(\nabla \bar{\xi}(\theta_0)\) is a subgradient. Note \(\bar{\xi}(\theta) - \bar{\xi}(\theta_0) = N^{-1} \sum_{i=1}^N (-\theta \cdot \tilde{x}_i) e_i(\theta) - N^{-1} \sum_{i=1}^N (-\theta_0 \cdot \tilde{x}_i) e_i(\theta_0) \), and \(\nabla\bar{\xi}(\theta_0) \cdot (\theta - \theta_0) = N^{-1}\sum_{i=1}^N (-\theta \cdot \tilde{x}_i) e_i(\theta_0) - N^{-1} \sum_{i=1}^N (-\theta_0 \cdot \tilde{x}_i) e_i(\theta_0)\). So \(\bar{\xi}(\theta) - \bar{\xi}(\theta_0) \geq \nabla\bar{\xi}(\theta_0) \cdot (\theta - \theta_0) \) if and only if \(\sum_{i=1}^N (-\theta \cdot \tilde{x}_i) e_i(\theta) \geq \sum_{i=1}^N (-\theta \cdot \tilde{x}_i) e_i(\theta_0) \). Consider the maximization problem \( \max_{t \in \R^{1+K}} \, \sum_{i=1}^N (-\theta \cdot \tilde{x}_i) e_i(t) \) where, recall, \(e_i(t) = y_i - \ind(t \cdot \tilde{x}_i > 0)\). When \(-\theta \cdot \tilde{x}_i > 0\), we want \(e_i(t)\) to be large, achieved when \(t \cdot \tilde{x}_i \leq 0\), and when \(-\theta \cdot \tilde{x}_i < 0\), we want \(t \cdot \tilde{x}_i > 0\). Clearly, this is achieved with \(t = \theta\). So the desired inequality holds for any \(\theta\) and \(\theta_0\), proving \(\nabla \bar{\xi}(\theta)\) is indeed a subgradient of \(\bar{\xi}(\theta)\) at any \(\theta\). Therefore, the Rosenblatt algorithm is the subgradient method (with a constant step size) that applied to the perceptron criterion.

    Hinge loss

    Problem of slack

    At a first glance, the perceptron criterion \(\eqref{eq:perceptron_criterion}\) appears to be a reasonable proxy for the empirical risk. However, the perceptron criterion suffers a technical flaw as an objective function: it is minimized at \(\theta = 0\) for any dataset.

    This property is harmless when the dataset is linearly separable. If we use \(\nabla \bar{\xi}(\theta) = -N^{-1} \tilde{X}^\mathsf{T} e(\theta)\) as the subgradient, then \(\nabla \bar{\xi}(0) \neq 0\) unless \(\theta = 0\) is a consistent hypothesis. So the subgradient method converges to a consistent hypothesis by the convergence theorem. When the dataset is not linearly separable, however, \(\theta = 0\) is the unique minimum, so the minimization problem is ill-posed.

    More fundamentally, the definition of slack is ill-suited for minimization. First, note that two different parameters associated with the same hypothesis have different average slacks. Recall that \(\theta\) and \(c \theta\) represent the same hypothesis if \(c > 0\), but \(\bar{\xi}(\theta) \neq \bar{\xi}(c \theta) = c \bar{\xi}(\theta)\).

    This isn't necessarily a problem per se. Since we are choosing the hypothesis by minimizing \(\bar{\xi}(\theta)\), each hypothesis is essentially represented by, among all the parameters associated with it, the one with the minimum average slack. But if the dataset is not linearly separable so that \(\bar{\xi}(\theta) > 0\) for any \(\theta\), then \(\min_{c > 0} \bar{\xi}(c \theta)\) does not exist since \(\bar{\xi}(c \theta) \to 0\) as \(c \to 0\). So under the minimum average slack criterion, each hypothesis lacks a representative parameter.

    Robust slack

    To fix the issue, instead of having \(\theta = 0\) always appear to be correct, we will have it always erroneous. More precisely, for some positive number \(\delta > 0\), we require \(\theta \cdot \tilde{x}_i \geq \delta\) when \(y_i = 1\) and \(\theta \cdot \tilde{x}_i \leq -\delta\) when \(y_i = 0\) for the classification to be considered correct. This leads to a "robust" error \[\begin{equation}\label{eq:robust_error} \tilde{e}_i(\theta; \delta) := \begin{cases} 1 & \text{if \(y_i = 1\) and \(\theta \cdot \tilde{x}_i < \delta\)}, \\ -1 & \text{if \(y_i = 0\) and \(\theta \cdot \tilde{x}_i > -\delta\)}, \\ 0 & \text{otherwise}, \end{cases} \end{equation} \] or, using \(\tilde{y}_i\) notation, \(\tilde{e}_i(\theta; \delta) = \tilde{y}_i \ind(\delta - \tilde{y}_i(\theta \cdot \tilde{x}_i) > 0)\). This robust error is always nonzero if \(\theta \cdot \tilde{x}_i \in (-\delta, \delta)\), and in particular, \(\tilde{e}_i(0; \delta) \neq 0\).

    The corresponding "robust" slack is \[ \tilde{\xi}_i(\theta; \delta) = \left( \delta \tilde{y}_i - \theta \cdot \tilde{x}_i \right) \tilde{e}_i(\theta; \delta) = \max\{0, \delta - \tilde{y}_i (\theta \cdot \tilde{x}_i)\}. \] Note \(\tilde{\xi}_i(\theta; 0) = \xi_i(\theta)\), so this version generalizes the previous one.

    plots the robust slack \(\tilde{\xi}\) against the signed score \(\tilde{y}_i (\theta \cdot \tilde{x}_i)\), i.e. the function \(f(x) = \max\{0, \delta - x\}\).

    Minimizing the average robust slack

    Finally, we can consider minimizing the average of robust slack, \[\begin{equation}\label{eq:average_slack}\bar{\xi}(\theta; \delta) := \frac{1}{N} \sum_{i=1}^N \tilde\xi_i(\theta; \delta).\end{equation}\] The corresponding surrogate loss with \(\delta > 0\) is called the hinge loss. It retains convexity and so is amenable to subgradient method, with \(\nabla \bar{\xi}(\theta; \delta) = -N^{-1} \tilde{X}^\mathsf{T}\tilde{e}(\theta; \delta)\) constituting a subgradient. Since \(\nabla\bar{\xi}(\theta; 0) \neq \nabla\bar{\xi}(\theta)\) at non-differentiable points, they choose different subgradients from the subdifferential.

    It also retains the convergence property. If the dataset is linearly separable, then the subgradient method converges to a consistent hypothesis, although not to a "borderline" one where \(\theta \cdot \tilde{x_i} = 0\) for at least one observation. Note that the value of \(\delta\), as long as positive, has no bearing. If a consistent hypothesis \(\theta\) is "interior," so that \(\theta \cdot \tilde{x}_i \neq 0\) for all observations but \(\tilde{e}_i(\theta; \delta) \neq 0\), then regardless of the value of \(\delta\), we can find \(c > 1\) such that \(\tilde{e}_i(c \theta; \delta) = 0\).

    Importantly, our initial grievance with the perceptron criterion is resolved: \(\bar{\xi}(0; \delta) = \delta > 0\). Recall that under the perceptron criterion, \(\bar{\xi}(c \theta; 0) = c \bar{\xi}(\theta; 0)\), so the minimizing parameter vanishes to zero.

    With \(\delta > 0\), the linearity no longer holds. However, \(f(c) := \bar{\xi}(c \theta; \delta)\) as a function of \(c\) is continuous and convex since each summand \(\tilde{\xi}_i(c\theta; \delta)\) is. Furthermore, \(f(0) = \delta\) and \(\lim_{c \to \infty} f(c) = \infty\) if at least one observation is misclassified. So if \(\inf_c f(c) < \delta\), then \(f(c)\) achieves the minimum at some \(c^* \in (0, \infty)\). plots this function \(f\) for the perceptron criterion, \(\delta = 0\), and the robust average slack, \(\delta > 0\). In short, a relevant hypothesis has a representative parameter under the robust slack.

    The value of \(\delta\) determines which of the associated parameters is representative, but we don't care about the choice of the parameter as long as there is one. As such, typically the value of \(\delta\) is fixed to unity \(\delta = 1\).

    Support vector machine

    The Rosenblatt algorithm finds a consistent linear threshold hypothesis when one exists and, with some technical modification, finds a hypothesis with a "good fit" when one does not. It is completely silent, however, on which hypothesis to choose when there are multiple consistent hypotheses. In short, it lacks an inductive bias.

    In this section, I define the margin to make the notion of wiggle room more precise. I then introduce a learning algorithm, the support vector machine (SVM), that selects the hypothesis that sits in the middle of the widest possible wiggle room. Lastly, I briefly discuss how this "natural belief" might be justified.

    Margin

    We want to be more precise about what it means for a separating hyperplane to lie in the middle of the wiggle room. We continue with the setting \(\mathcal{X} = \R^K\) and \(\mathcal{Y} = \{0, 1\}\).

    Let \(\theta = (\theta_0, \dots, \theta_K)\) be a parameter vector with bias \(b_\theta := \theta_0\) and nonzero linear score \(w_\theta := (\theta_1, \dots, \theta_K) \neq 0\). Denote the hyperplane corresponding to \(\theta\) as \[ \ell(\theta) := \{x \in \R^K : \theta \cdot \tilde{x} = b_\theta + w_\theta \cdot x = 0\}. \]

    Given a dataset \((X, y)\), define the margin of \(\theta\) as the distance between the hyperplane \(\ell(\theta)\) and the closest input observation: \[ \min_i \frac{|b_\theta + w_\theta \cdot x_i|}{\lVert w_\theta \rVert} = \min_i \frac{|\theta \cdot \tilde{x}_i|}{\lVert w_\theta \rVert}, \] and call \(x_i\) a support vector if \(i\) attains this minimum. Since this quantity is homogeneous of degree zero, parameter vectors representing the same hyperplane have the same margin.

    Hard margin

    Our motivation was to have a selection criterion when there are multiple consistent hypotheses, so let's suppose the dataset is linearly separable. Our inductive bias prefers a hypothesis with a larger margin, so the learning algorithm chooses the hypothesis with the largest margin amongst all the consistent ones: \[\begin{align*} \max_{\theta \in \R^{1+K}} \quad &\lVert w_\theta \rVert^{-1} \left(\min_{x_i \in X} \; |\theta \cdot \tilde{x}_i| \right) \\ \text{s.t.} \quad &e_i(\theta) = 0 \text{ for all \(i=1,\dots,N\)}. \end{align*} \] This learning algorithm is called the hard margin SVM because it imposes the hard constraint of no slack.

    The problem can be simplified with some normalization. Denote \(\delta := \min_{x_i \in X} \; |\theta \cdot \tilde{x}_i|\). The hyperplane with the maximal margin will have a positive margin (by the "wiggle room" argument), so \(\delta > 0\). Asking the absolute affine score to be at least \(\delta > 0\) is precisely what we did for the robust error \(\eqref{eq:robust_error}\). That is, \(|\theta \cdot \tilde{x}_i| \geq \delta\) together with \(e_i(\theta) = 0\) translate to \(\tilde{e}_i(\theta; \delta) = 0\).

    So, the margin maximization can be written as \[\begin{equation}\label{eq:svm_hard}\begin{aligned} \min_{\theta \in \R^{1+K}} \quad &\frac{1}{2}\lVert w_\theta \rVert^2 \\ \text{s.t.} \quad &\tilde{e}_i(\theta; \delta) = 0 \text{ for all \(i=1,\dots,N\)}. \end{aligned} \end{equation}\] Here we recognized that maximizing \(\lVert w_\theta \rVert^{-1}\) is equivalent to minimizing \(\frac{1}{2} \lVert w_\theta \rVert^{2}\), where the squaring and halving are applied purely for the cleaner gradient expression. Recalling \(\tilde{e}_i(\theta; \delta) = \tilde{y}_i \ind(\delta - \tilde{y}_i(\theta \cdot \tilde{x}_i) > 0)\), the condition \(\tilde{e}_i(\theta; \delta) = 0\) is equivalent to \(\tilde{y}_i(\theta \cdot \tilde{x}_i) \geq \delta \). Again, the value of \(\delta\) determines the "representative" parameter of the hypothesis and is often fixed to \(\delta = 1\).

    Soft margin

    The "hard" constraint of the minimization problem \eqref{eq:svm_hard} can be satisfied if and only if the dataset is linearly separable. What if the dataset is not linearly separable? We've already seen that we can instead look for the hypothesis that minimizes the average slack.

    But now we have two competing criteria for hypothesis selection: how well the hypothesis fits the data, measured by the average slack \(\eqref{eq:average_slack}\), and how well it fits the inductive bias, measured by the margin.

    A simple solution is to minimize the weighted sum of each, yielding the following optimization problem: \[\begin{equation}\label{eq:svm} \min_{\theta \in \R^{1+K}} \; \bar{\xi}(\theta; \delta) + \lambda \frac{1}{2} \lVert w_\theta \rVert^2, \end{equation} \] where \(\lambda \geq 0\) determines how much we care about maximizing the margin over the data fit. A learning algorithm that chooses the argument of the minimum of \(\eqref{eq:svm}\) is called the (soft margin) support vector machine (SVM).

    Since the objective function \(f(\theta) := \bar{\xi}(\theta; \delta) + \lambda \frac{1}{2} \lVert w_\theta \rVert^2\) is convex, a subgradient method remains a suitable optimization strategy (Shalev-Shwartz et al., 2011). From our earlier derivations with the hinge loss, we can characterize a subgradient as \[\begin{align*} \nabla_{w_\theta} f(\theta) &= -\frac{1}{N} X^\mathsf{T} \tilde{e}(\theta; \delta) + \lambda w_\theta, \\ \pdv{f(\theta)}{b_\theta} &= -\frac{1}{N} \sum_{i=1}^N \tilde{e}_i(\theta; \delta). \end{align*}\]

    A lower value of \(\lambda\) corresponds to higher emphasis on data-fitting, with the algorithm reducing to the (corrected) Rosenblatt when \(\lambda = 0\). A typical explanation goes: When \(\lambda\) is too low, the algorithm "overfits" to the given (training) dataset. Conversely, when \(\lambda\) is too high, it "underfits" the dataset. So there is a Goldilocks \(\lambda\) that balances the data-fitting and the inductive bias to minimize the true risk. Finding the right \(\lambda\) itself poses an optimization problem.

    Justifying maximal margin

    Is maximal margin a good inductive bias? One strand of argument, summarized for example by Vapnik (1999), notes that the set of hypotheses with a larger margin has a lower VC dimension than the set of hypotheses with a smaller margin. Since an upper bound on the risk increases with the VC dimension, choosing the hypothesis with the maximal margin minimizes this bound and thus promotes good generalization.

    A couple of issues with this argument. For one, the margin of a hypothesis is determined in conjunction with a given dataset, whereas the upper bound involves the VC dimension of the hypothesis space \(H\), which is defined independently of the dataset. (By this reasoning, one could always pick a hypothesis \(h\) and claim that it generalizes well since the singleton space \(\{h\}\) has low VC dimension!) Mount (2015) provides a nice summary on this issue, which has largely been addressed by extensions of VC dimension. At the frontier of the extension work seems to be Rademacher complexity. See, for example, Koltchinskii and Panchenko (2000) and Bartlett and Mendelson (2002).

    Another issue is that minimizing an upper bound of something does not translate to minimizing the thing. The literature's view on this gap can feel quite harsh. Works such as Zhang et al. (2017), Nagarajan and Kolter (2019), and Gastpar et al. (2024) cast doubt on the usefulness of any bounds based on the uniform convergence approach.

    A different strand of argument, which I think better captures the intuitive appeal of the maximal-margin classifier, notes that observed data may be perturbed with noise. If so, it would be sensible to choose a classifier that is far away from individual observations to avoid contamination. Xu et al. (2009) provide a rigorous version of this argument.

    Taking stock

    We have considered the hypothesis space generated by the linear threshold function \(\eqref{eq:ltf}\). We started by noticing that the Rosenblatt algorithm can be seen as following the principle of ERM if we are willing to use a surrogate loss function to make the problem convex. Doing so allowed us to notice a technical problem with the objective function which we fixed by introducing the hinge loss. Finally, we described the SVM as incorporating an inductive bias in the form of maximal margin.

    The examples of perceptron so far have had a fairly restricted type of the hypothesis space. All the iterations have been on the learning algorithm alone. Time to examine the role of hypothesis space.

    Minsky & Papert perceptron

    The definition of perceptron given by Minsky and Papert (1969), henceforth MP, applies to the binary output space \(\mathcal{Y} = \{0, 1\}\) and takes the hypothesis generating function to be \[\begin{equation}\label{eq:mp_perceptron} h(x, \theta) = \ind \left( \theta \cdot \varphi(x) > 0\right), \end{equation}\] where \(\varphi: \mathcal{X} \to \R^L\) and \(\theta \in \R^L\) for some positive integer \(L\). I will call

    • the function \(\varphi\) the feature map,
    • the \(L\)-vector \(\varphi(x)\) the feature vector, and
    • the codomain of the feature map, \(\R^L\), the feature space. More generally, the feature space can be any inner product space. In the book, MP restricts the feature space to be \(\{0,1\}^L\).

    Clearly, the linear threshold hypothesis generating function \(\eqref{eq:ltf}\) is a special case with the feature map \(\varphi(x) = (1, x_1, \dots, x_K)\). Conversely, one could reasonably see the MP perceptron as a simple perceptron with processed input data, in which case the generalization does not gain us much. I think, however, that folding the feature map into the perceptron rather than the dataset provides a useful framing. As the §kernel SVMs show, applying features does not require computing the features prior to executing the learning algorithm. Thus, "data processing" can be a misleading interpretation.

    Because the definition of MP perceptron imposes no restriction on the feature map, it allows quite a broad range of hypothesis spaces. For example, if \(\mathcal{X}\) is finite, then there exists an MP perceptron whose hypothesis space is \(F(\mathcal{X}, \mathcal{Y})\). Indeed, one can choose the feature map \(\varphi\) so that each component function \(\varphi_\ell\) corresponds to a function in \(F(\mathcal{X}, \mathcal{Y})\). Then for any \(f^* \in F(\mathcal{X}, \mathcal{Y})\), we can set the corresponding coefficient to a positive number, say 1, and all other coefficients to zero. The resulting hypothesis is identical to \(f^*\) since \(\ind(f^*(x) > 0) = f^*(x)\). This result is Theorem 1 of Rosenblatt (1962), which he acknowledges to be "of little practical interest." MP likewise reiterate that such a feature map is "impossible in practice" since the feature space dimension "would have to be astronomically large." These texts suggest that, at the time, the main motivation for restricting the hypothesis space was computational feasibility rather than providing inductive bias.

    Locality

    MP's main agenda is to examine what functions are excluded from the hypothesis space when certain restrictions are imposed on the feature map. Given \(\varphi(x) = (\varphi_1(x), \dots, \varphi_L(x))\), these restrictions take the form of "locality" conditions on the component functions \(\varphi_\ell\).

    Order restriction

    One example of locality is what MP call order restriction. An MP perceptron is of order \(\boldsymbol{n}\) if each component function \(\varphi_\ell\) depends on at most \(n\) coordinates of \(x \in \mathcal{X}\). The simple perceptron, for instance, is an MP perceptron of order 1.

    A popular example considers order 1 perceptrons for binary operators. Suppose \(\mathcal{X} = \{0, 1\}^2\), so that \(F(\mathcal{X}, \mathcal{Y})\) consists of all 16 binary Boolean operators. Then the simple perceptron hypothesis space excludes two operators: XOR, \(\ind(x_1 \neq x_2)\), and its negation, XNOR, \(\ind(x_1 = x_2)\). To prove that XOR is excluded from the order 1 hypothesis space, we can show that there is no \((\theta_0, \theta_1, \theta_2) \in \R^3\) such that \(h(x_1, x_2) = \ind(\theta_0 + \theta_1 x_1 + \theta_2 x_2 > 0)\) is the XOR function, i.e. \(h(0, 0) = h(1, 1) = 0\) and \(h(0, 1) = h(1, 0) = 1\). In other words, we would need the following system to hold: \(-\theta_0 \geq 0\), \(\theta_1 + \theta_2 \leq -\theta_0\), \(\theta_1 > -\theta_0\), and \(\theta_2 > -\theta_0\). Indeed, since \(-\theta_0\) must be non-negative, and \(\theta_1\) and \(\theta_2\) are each greater than \(-\theta_0\), their sum must be greater than \(-\theta_0\), showing the system has no solution.

    Diameter restriction

    Another example of locality is diameter restriction, which requires each component function \(\varphi_\ell\) of the feature map to depend only on a set of points whose diameter is no greater than a pre-specified threshold. , reproduced from MP, illustrates this restriction.

    Responses

    The central result of MP is that the hypothesis spaces generated by locality restrictions on the feature map, such as finite order or diameter, exclude many predicates, such as connectedness. The implied conclusion is that perceptrons in general are fundamentally limited in their capabilities and cannot by themselves attain human-like intelligence. The book eagerly frames itself as a neutral investigation rather than a criticism of the perceptron: "It is no more apt to describe our mathematical theorems as pessimistic than it would be to say the same about deducing the conservation of momentum from the laws of mechanics." It is no secret, however, that the authors took a critical stance (Olazaran, 1996). In the epilogue of the expanded edition, MP write: "Our research on perceptrons and on other computational schemes has left us with a pervasive bias against seeking a general, domain-independent theory of 'how neural networks work.'"

    A critical response to the result questions two aspects of it. One is the nature of the excluded predicates, and the other is the nature of the locality restriction.

    Should perceptrons detect connectedness?

    It is a tautology that if \(f^* \notin H\), then \(\phi(D(X, f^*)) \neq f^*\) for any learning algorithm \(\phi\) and input dataset \(X\). Or, as MP put it, "No machine can learn to recognize X unless it possesses, at least potentially, some scheme for representing X." So the observation that connectedness is excluded is meaningful only if we think such predicates are important in some sense. MP's implication seems to be that connectedness is an "easy" task that any learning machine ought to be able to do. (They make the point by including a "serial" algorithm for detecting connectedness.)

    But detecting connectedness and other properties such as symmetry is not as easy as it may first seem. Humans also struggle with such tasks when the image is sufficiently large and complex. Given the goal of mimicking human intelligence, failure to reliably detect connectedness for arbitrarily large images may not be a major deficiency.

    MP push back in the expanded edition, arguing that humans can detect connectedness and symmetry even for large and complex images, just not at first sight as we can for smaller images. We adopt a different mode of thinking, one that is slower and more deliberate, and often reach the correct answer.

    Is locality a reasonable restriction?

    MP's formal results focus on "locality" constraints, primarily finite order and diameter. The main justifications for such restrictions are analytical tractability and computational feasibility. (Notions of inductive bias and generalization are absent in MP.) But by the time MP was published, perceptron researchers had already been working on "non-local" feature maps in the form of multilayered perceptrons — MP perceptrons whose component functions \(\varphi_\ell\) are themselves MP perceptrons, i.e. each takes the form \(\eqref{eq:mp_perceptron}\) for some parameter \(\theta\).

    While the formal results do not apply to multilayer perceptrons, MP do not hesitate to offer pessimistic conjectures:

    There is no reason to suppose that any of these virtues [of single-layered perceptrons] carry over to the many-layered version. Nevertheless, we consider it to be an important research problem to elucidate (or reject) our intuitive judgment that the extension is sterile.

    Safe to say, the multilayer extension proved to be far from sterile, and even a cursory look at the topic would require a separate note. In MP's defense, they expected multilayer perceptrons to reach a computational bottleneck, one overcome primarily by the growth of computational power rather than by algorithmic efficiency. Olazaran (1996) includes an interview with Minsky. Regarding "intuitive judgment", Minsky said: "Now, 'efficiently' in the 1960s meant a few thousand trials. Of course now if it does it in a million trials it is not so bad."

    Kernel SVM

    Multilayer perceptrons are not the only way to feasibly relax the locality constraint on the feature map. SVMs, too, can accommodate arbitrarily complex feature maps through a clever trick.

    To illustrate the need for the trick in the first place, suppose we want a feature map consisting of all the 4th-degree polynomial terms of the MNIST inputs. Since each MNIST input is a 784-vector, this would yield a feature vector of length \({784 + 4 \choose 4}\), about 16 billion altogether. If we used a subgradient method to solve the SVM problem, each update would require calculating dot products of 16-billion-dimensional vectors.

    Two observations in tandem help us solve this computational problem.

    1. The representer theorem tells us that the solution to the SVM problem lies in the span of the input data, meaning \(w^* = \sum_{i=1}^N c_i x_i\) for some coefficient vector \(c = (c_1, \dots, c_N) \in \R^N\). So we can solve for the \(N\)-dimensional \(c\) instead of the \(K\)-dimensional \(w\) directly.
    2. If a function \(k : \mathcal{X}^2 \to \R\), called a (Mercer) kernel, satisfies \(k(x_i, x_j) = \varphi(x_i) \cdot \varphi(x_j)\) for some feature map \(\varphi\), then we can replace \(x_i \cdot x_j\) with \(k(x_i, x_j)\) and work with the expanded feature space without explicitly computing the feature vectors.

    The representer theorem

    To state the first observation cleanly, let us temporarily return to the linear SVM setting and suppress the feature map. The representer theorem tells us that the solution to the SVM problem lies in the span of the input data, meaning \(w^* = \sum_{i=1}^N c_i x_i\) for some coefficient vector \(c = (c_1, \dots, c_N) \in \R^N\). This allows us to solve for the \(N\)-dimensional \(c\) instead of the \(K\)-dimensional \(w\) directly.

    To see this result, consider solving the SVM minimization problem \(\eqref{eq:svm}\) by a subgradient method. For now, let's fix the bias term \(b_\theta\) and minimize the objective over the weights \(w_\theta\). It is useful to define \[\begin{align*} s_i(w) &:= \delta - \tilde{y}_i (b_\theta + w \cdot x_i), \\ \tilde{e}_i(w) &:= \tilde{y}_i \ind(s_i(w) > 0), \\ \tilde{\xi}_i(w) &:= \max\{0, s_i(w)\}, \end{align*}\] so that we can write the SVM objective as a function of \(w\): \[\begin{equation}\label{eq:svm_objective_w} f(w) := \frac{1}{N} \sum_{i=1}^N \tilde{\xi}_i(w) + \lambda \frac{1}{2} \lVert w \rVert^2. \end{equation}\]

    For any given initial guess \(w_0 \in \R^K\) and step size sequence \((\eta_0, \eta_1, \eta_2, \dots)\), a subgradient method produces a sequence \((w_0, w_1, w_2, \dots)\) by \[w_{t+1} = w_t - \eta_t \nabla f(w_t)\] for all \(t = 0, 1, 2, \dots\), where \(\nabla f(w_t)\) is a subgradient of \(f\) evaluated at \(w_t\). One valid choice is \[\begin{equation}\label{eq:svm_subgradient} \nabla f(w_t) = - \frac{1}{N} X^\mathsf{T} \tilde{e}(w_t) + \lambda w_t. \end{equation}\] With large \(K\), evaluating this subgradient presents a computational challenge because it requires the dot products \(w_t \cdot x_i\) at each update step.

    Notice, however, that if we start from the zero vector \(w_0 = 0\), then \(w_1 = \eta_0 \frac{1}{N} \sum_{i=1}^N \tilde{e}_i(w_0) x_i\) is a linear combination of the input data \(\{x_1, \dots, x_N\}\). The same is true of \(w_2\), which is a weighted sum of \(w_1\) and \(\sum_{i=1}^N \tilde{e}_i(w_1) x_i\). By induction, every member of the sequence \((w_1, w_2, \dots)\) is a linear combination of the input data. That is, \(w_t \in \span\{x_1, \dots, x_N\}\) for all \(t = 1, 2, \dots\).

    With the constraint \(w_t = \sum_{i=1}^N c_{i,t} x_i = X^\mathsf{T} c_t\) for some coefficient vector \(c_t = (c_{1,t}, \dots, c_{N,t}) \in \R^N\), we have \[ w_t \cdot x_i = \sum_{j=1}^N c_{j,t} (x_j \cdot x_i) = (Gc_t)_i, \] where \(G := XX^\mathsf{T}\) is the Gram matrix of the input data \(x_1, \dots, x_N\), meaning its \(ij\)-th entry is \(x_i \cdot x_j\). This already eases the computational burden. Since the dot products \(x_j \cdot x_i\) are invariant across steps \(t\), we can compute them once at the beginning rather than at every update step.

    If the input space dimension is in the billions or higher, computing the dot product even once may not be feasible. But if that high dimensionality comes from a feature map, as in our motivating example of fourth-degree polynomial expansion, then there may be ways to compute the corresponding dot product much more easily. Before we discuss that, however, let's return to the subgradient method and push the observation a little further.

    We've established that starting from the zero vector \(w_0 = 0\), each update \(w_t\) produced by the subgradient method belongs to the span of the dataset \(\span\{x_1, \dots, x_N\}\), an at-most-\(N\)-dimensional subspace. Since the objective \(\eqref{eq:svm_objective_w}\) is strictly convex if \(\lambda > 0\), there is a unique minimizer \(w^*\), and with an appropriate step size sequence \((\eta_t)\), the subgradient method converges to it, \(w_t \to w^*\).

    Since the span is a finite-dimensional subspace, it is closed. So the limit of the sequence \((w_t)\), namely the minimizer \(w^*\), also belongs to the span. In other words, the minimizer of the SVM satisfies \(w^* = \sum_{i=1}^N c_i x_i\) for some coefficients \((c_1, \dots, c_N) \in \R^N\).

    The result holds for a more general class of objective functions and is known as the representer theorem (Kimeldorf & Wahba, 1970; Schölkopf et al., 2001). Here's a simplified version of the proof for the SVM objective in a finite-dimensional inner product space. Let \(V = \span\{x_1, \dots, x_N\}\). Then \(\R^K = V \oplus V^\perp\) where \(V^\perp\) is the orthogonal complement of \(V\). This means that for any \(w \in \R^K\), there exists a unique representation \(w = v + u\) where \(v \in V\) and \(u \in V^\perp\). Note that the hinge loss term \(\tilde\xi_i(w)\) depends on \(w\) only through \(w \cdot x_i = (v + u) \cdot x_i = v \cdot x_i\), where the last equality follows from \(u \in V^\perp\). So the hinge loss depends on \(w\) only through \(v\), its projection onto \(V\). Furthermore, by the Pythagorean theorem, \(\lVert w \rVert^2 = \lVert v \rVert^2 + \lVert u \rVert^2\), so it is minimized when \(u = 0\). Therefore, if \(w^*\) is a minimizer, then \(w^* \in V\).

    Mercer kernels

    Let's return to the computational problem we had: calculating the dot product \(\varphi(x) \cdot \varphi(x')\) when the feature space is high-dimensional, i.e. \(\varphi : \mathcal{X} \to \R^L\) with large \(L\). For a simple illustration, suppose the input space is \(\mathcal{X} = \R^2\), and the feature space contains all quadratic terms. Denote two input observations by \(x = (a, b)\) and \(x' = (a', b')\), so we can write the feature map as \(\varphi(x) = (a^2, b^2, \sqrt{2}ab, \sqrt{2}a, \sqrt{2}b, 1)\). Why the \(\sqrt{2}\) scaling? Notice \[\begin{align*} (x \cdot x' + 1)^2 &= (aa' + bb' + 1)^2 \\ &= (aa')^2 + (bb')^2 + 2aa'bb' + 2aa' + 2bb' + 1 \\ &= \varphi(x) \cdot \varphi(x'). \end{align*}\] So we can compute the dot product in this 6-dimensional feature space from the dot product of the 2-dimensional inputs using the function \(k(x, x') := (x \cdot x' + 1)^2\).

    More generally, for higher-degree polynomial feature maps, we can use the function \[ k(x, x') = (x \cdot x' + c)^d \] where \(c \geq 0\) is a constant and \(d\) is the polynomial degree. So, if we wanted a fourth-degree polynomial feature map as in our motivating example, we could use, say, \(k(x, x') = (x \cdot x' + 1)^4\).

    It is natural to ask at this point: what about other feature spaces? But this leads to a treacherous path. One issue is that we first need to clarify what we want from a function \(k\) beyond the identity \(k(x, x') = \varphi(x) \cdot \varphi(x')\). Roughly, we want \(k\) to be computationally simple, but we have already seen how difficult it is to make simplicity precise. More importantly, we do not really care about a specific feature space. We just want one broad enough to capture the data generating function.

    So, instead of committing to a feature map \(\varphi\) and then looking for a "simple" function \(k\) that computes the dot product, we ask the reverse: what are the properties of a function \(k : \mathcal{X}^2 \to \R\) such that \(k(x, x') = \varphi(x) \cdot \varphi(x')\) for some feature map \(\varphi\)?

    For one, since dot product is symmetric (in real spaces), \(\varphi(x) \cdot \varphi(x') = \varphi(x') \cdot \varphi(x)\), the function \(k\) must also be symmetric, \(k(x, x') = k(x', x)\). More generally, for any dataset \(X = (x_1, \dots, x_N)\), the matrix \(K\) whose \(ij\)-th entry is \(k(x_i, x_j)\) must be the Gram matrix of \(\varphi(x_1), \dots, \varphi(x_N)\). But a matrix is a Gram matrix if and only if it is positive semidefinite. This leads to the notion of a Mercer kernel: A function \(k : \mathcal{X}^2 \to \R\) is a (Mercer) kernel if the matrix \(K\), whose \(ij\)-th entry is \(k(x_i, x_j)\), is positive semidefinite for any dataset \(X \in \mathcal{X}^N\).A covariance matrix is positive semidefinite, and any positive semidefinite matrix is a covariance matrix of some random variables. So the term kernel function is used synonymously with covariance function.

    This definition also gives us a way to build new kernels from existing ones. For example, since sums and entrywise products of positive semidefinite matrices are positive semidefinite, sums, products, and powers of kernels remain kernels. From this, it is easy to see that \(k(x, x') = (x \cdot x' + c)^d\) is a kernel for \(c \geq 0\).

    If we trust that maximal margin provides a sufficiently good inductive bias, then it is natural to seek kernels that make the hypothesis space as broad as computationally feasible. A popular choice is the Gaussian or radial basis function (RBF) kernel: \[k(x, x') = \exp(-\gamma\lVert x - x'\rVert^2)\] with parameter \(\gamma > 0\). Roughly speaking, it corresponds to a feature map with polynomial terms of all degrees, so its associated feature space is infinite-dimensional. Indeed, \[ \exp\left(-\gamma \lVert x - x' \rVert^2\right) = \exp\left(-\gamma \lVert x \rVert^2\right) \exp\left(-\gamma \lVert x' \rVert^2\right) \exp\left(2\gamma x \cdot x'\right). \] Since \(\exp(t) = \sum_{m=0}^\infty t^m / m!\), the linear expansion of the last term includes powers of \(x \cdot x'\) of every degree.

    In the simplified case where the bias is fixed to zero, one implementation of the kernel SVM procedure is as follows.

    1. Given
      • a dataset \((X, \tilde{y})\), where \(X = (x_1, \dots, x_N)\) and \(\tilde{y}_i \in \{-1, 1\}\), and
      • a kernel \(k : \mathcal{X}^2 \to \R\),
    2. Choose
      • the initial coefficient vector \(c_0 \in \R^N\),
      • the weight on the margin term \(\lambda > 0\),
      • the length of the sequence \(T > 0\), and
      • the step-size sequence \((\eta_1, \dots, \eta_T)\), with each \(\eta_t > 0\),
    3. Compute the Gram matrix \(K \in \R^{N \times N}\) with \(K_{ij} = k(x_i, x_j)\).
    4. Recursively define the sequence \((c_1, \dots, c_T)\) by \[ c_{t+1} := c_t + \eta_{t+1}\left(\frac{1}{N}\tilde{e}(c_t) - \lambda c_t\right), \] where \[ \tilde{e}_i(c_t) := \tilde{y}_i \ind\left(1 - \tilde{y}_i (Kc_t)_i > 0\right), \quad i = 1, \dots, N. \]
    5. Return \(c_T\). The predicted output value can be computed as \[ \phi(x) = \ind\left(\sum_{i=1}^N c_{i,T} k(x_i, x) > 0\right). \]

    Example: Kernel SVM on MNIST

    To compare with the earlier linear SVM, I again used the MNIST dataset restricted to digits 4 and 9, and fit SVMs with several kernels over a range of \(\lambda\) values. Compared with the earlier linear SVM results, the kernelized versions achieved significantly lower test error: the best kernel runs were around \(0.8\%\) to \(1.0\%\), whereas the best linear SVM result was about \(2.8\%\).

    Kernel SVMs thus show that a broad feature space need not be computationally prohibitive. We do not need to explicitly construct the feature map, so long as we can compute the corresponding dot products through a kernel function. In this way, kernels offer another route around the locality restrictions emphasized by MP.

    TLDR

    1. A learning machine tries to infer the relationship between input and output variables from sample observations of input-output pairs. That is a difficult problem. To make meaningful inferences, you need some prior — a sense of which relationships are more likely than others before you observe anything.
    2. A perceptron is a learning machine endowed with two components: the hypothesis space, the set of candidate relationships it can choose from, and the learning algorithm, a rule for picking one hypothesis. This is a much broader description than what someone today would usually mean by perceptron, but I think it fairly reflects what Rosenblatt had in mind when he coined the term.
    3. Why restrict the hypothesis space in the first place? Because doing so lets you rule out relationships you regard as unlikely. Yes, that is a blunt way to impose a prior. Minsky and Papert wrote a whole book about just how blunt it is.
    4. If the observations reflect the input-output relationship accurately, then the candidate hypothesis should at least match the observed input-output pairs. More pompously, you can say the learning algorithm should minimize the empirical risk.
    5. Historically, people have closely studied the case where the output is binary and each hypothesis takes a weighted sum of input values and checks whether the sum exceeds some threshold. Rosenblatt's original algorithm minimizes a version of empirical risk in this setting.
    6. Empirical risk minimization alone does not help you infer beyond what you observe. Supplement the Rosenblatt algorithm with an additional prior, namely a preference for higher-margin hypotheses, and you get a support vector machine. It picks a hypothesis that fits both the observations and the prior well.
    7. Modern perceptrons can work with much larger hypothesis spaces (in part through the growth of raw computational power). One example is the support vector machine with the so-called kernel trick. Another is the multilayer perceptron, or neural network, which is the topic of my next study note.
